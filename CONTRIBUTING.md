# Contributing

Contributions are welcome — bug fixes, improvements to the layout, documentation clarifications, or new features that keep the template minimal.

## How to contribute

1. Fork the repo
2. Create a branch: `git checkout -b your-feature`
3. Make your changes and test locally with `hugo serve`
4. Commit with a clear message
5. Open a pull request describing what you changed and why

## Guidelines

- Keep the template minimal — avoid adding JavaScript frameworks or heavy dependencies
- Test with Hugo Extended v0.120+
- If you change CSS, verify the layout on mobile and desktop
- If you change Hugo templates, verify the site builds without warnings (`hugo --verbose`)

## Reporting issues

Open a GitHub issue with:
- Hugo version (`hugo version`)
- What you expected vs. what happened
- Steps to reproduce
